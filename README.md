# 포켓몬 앱

간단한 Android 앱 프로젝트입니다. 최신 Android 기술 스택을 사용하여 효율적이고 구조화된 앱 개발을 목표로 합니다.

본 앱은 포켓몬 리스트를 제공하며, 각 캐릭터별 상세 정보를 확인할 수 있습니다.  
사용자는 포켓몬을 이름(영어)으로 검색할 수 있고, 원하는 포켓몬을 즐겨찾기(좋아요)하여 손쉽게 관리할 수 있습니다.  
검색과 좋아요 기능을 통해 더욱 편리하고 개인화된 포켓몬 탐색 경험을 제공합니다.


## 기술 스택

- **MVVM 패턴**  
  UI와 비즈니스 로직을 분리하여 코드의 유지보수성과 테스트 용이성을 높이는 아키텍처 패턴.

- **Flow**  
  코틀린 코루틴 기반의 비동기 데이터 스트림 처리 라이브러리로, 연속적인 데이터 흐름을 안전하게 처리.

- **Coroutines**  
  경량화된 비동기 프로그래밍을 지원하여 복잡한 비동기 작업을 간결하고 명확하게 구현.

- **Hilt**  
  안드로이드에 특화된 의존성 주입(DI) 라이브러리로, 컴파일 타임에 안전하게 객체 의존성을 관리.

- **Mapper**  
  데이터 모델 간 변환을 담당하여, 외부 데이터와 도메인 모델 간 일관된 매핑을 지원.

- **Clean Architecture**  
  계층별 책임을 명확히 분리하여 확장성 및 테스트 편의성을 극대화하는 소프트웨어 아키텍처.

- **Compose**  
  선언형 UI 프레임워크로, 코드로 UI를 직접 구성해 생산성을 높이고 유지보수를 쉽게 함.

- **Coil**  
  Kotlin으로 작성된 경량 이미지 로딩 라이브러리로, Compose 등 최신 UI 프레임워크와 호환성이 높음.

- **Room**  
  SQLite 데이터베이스를 추상화해 간편한 데이터 저장 및 조회 기능을 제공하는 Android Jetpack 라이브러리.


## 개발 환경
- Android Studio 최신 버전
- Kotlin 1.8+
- minSdkVersion: 26
- targetSdkVersion: 34


## Android 프로젝트 폴더 구조

| 폴더 경로                    | 역할 및 설명                                                         |
|------------------------------|---------------------------------------------------------------------|
| `data/api`                   | 서버와 통신하는 API 인터페이스 및 구현체                            |
| `data/database`              | 로컬 데이터베이스 관련 코드 (Room 등)                               |
| `data/model`                 | 네트워크 및 DB와 연동되는 데이터 모델 정의                          |
| `data/repository`            | 데이터 소스 관리와 추상화, 데이터의 흐름 제어                       |
| `di/module`                  | Hilt 등 의존성 주입 모듈 정의 및 구성                               |
| `domain/mapper`              | 데이터 변환기 (외부 모델 ↔ 도메인 모델)                            |
| `domain/model`               | 비즈니스 로직 중심의 도메인 모델 정의                              |
| `domain/repository`          | 도메인 레이어에서 데이터 소스 추상화 인터페이스 정의              |
| `domain/usecase`             | 비즈니스 로직 단위인 유스케이스 구현                               |
| `presentation/component`     | UI의 재사용 가능한 컴포넌트 구현                                    |
| `presentation/screen`        | 화면 단위 UI 구성 및 화면 전환 로직                                 |
| `presentation/viewmodel`     | UI와 도메인 계층 연결, 상태 관리와 로직 처리                        |
| `utils`                     | 프로젝트 전반에서 사용되는 유틸리티 함수 및 헬퍼                    |
